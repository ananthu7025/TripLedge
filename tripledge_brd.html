1. Overview

Tripledge is a field operations mobile app with two core record types: Snow (snow removal jobs) and Trip (trip hazard inspections). Each record type has two sequential phases — Inspection and Completion. An Admin panel supports map management, job tracking, and user performance analytics.

ModuleRecord TypePhasesKey Output
SnowSnow JobInspection → CompletionBefore/after photos, issue resolution log, crew record
TripTrip InspectionInspection → CompletionMeasurements (H/L/Length), GPS coordinates, photo evidence
Admin——Map zones, job dashboard, user performance reports

8. Trip Scoring Matrix

Scoring Logic

A single numeric Trip Score per record rewards both length and difficulty of work. Two workers with the same total length receive different scores if one dealt with a steeper height differential.

Core Formula — per Trip Record
Difficulty = High Point − Low Point
Trip Score = Length × (1 + Difficulty × Weight)

// Weight is a tunable constant (recommend starting at 0.1 – 0.5)
// All measurements in same unit (cm recommended)
Rationale: A user with Length=5, Difficulty=8 scores higher than a user with Length=5, Difficulty=2 — even though both covered the same distance. The Weight constant allows admins to tune how much difficulty is rewarded vs raw length.

Example Comparison

UserLengthHighLowDifficultyScore (W=0.2)
User A510285 × (1 + 8 × 0.2) = 13.0
User B55325 × (1 + 2 × 0.2) = 7.0
User C412394 × (1 + 9 × 0.2) = 11.2

User A wins despite same length as User B. User C (shorter length, harder job) ranks between them.

Aggregated User Score

Cumulative Score — across all trips for a user
Total Score = Σ ( Length_i × (1 + Difficulty_i × Weight) )
Config note: The Weight constant should be configurable by the Admin (not hard-coded). Recommend exposing it in Admin → Settings.

9.3 Map Capabilities Summary

  • Add unlimited named locations with auto-incremented naming
  • Draw and store a polygon boundary per location
  • Assign type (Proposed / Additional) with colour-coded map rendering
  • Edit or delete existing locations from map
  • Trip inspections can be linked to a location zone (for admin filtering)
